# Framework: Multi-Variant Draft

Multi-Variant Draft produces 2 to 3 alternative versions of the same email with meaningfully different angles. Use it ONLY when the brief or the flow explicitly calls for variants: an A/B test request, a "give me a long version and a short version" ask, a flow-node draft that needs detailed and concise variants for the brand to choose between. Do not use it as the default. Do not use it to hedge between angles you cannot decide on. Playbook rule: each variant is internally complete (own greeting, own CTA). The objective stays constant; the ANGLES differ meaningfully (detailed vs concise, founder voice vs brand voice, soft vs urgent). If the variants differ only in word choice, the framework adds no value.

## Structural rules

- Setup: optional short header before the variants if context helps the brand reviewer. Skip when the variants speak for themselves.
- Variant labels: explicit. Each variant gets a clear label naming its angle: "**Variant A (Long)**", "**Variant B (Short)**", "**Variant A (Founder Voice)**", "**Variant B (Brand Voice)**".
- Each variant complete: own greeting, own body, own CTA, own sign-off. The variant must work as a standalone email.
- Same destination across variants: the CTA points at the same URL unless the brief calls for divergent destinations. The angles differ; the action does not.
- Optional Variant C: only when the brief explicitly calls for three. Two variants is the default.
- Common close: one shared unsubscribe footer at the bottom is acceptable; do not duplicate it under each variant.
- Do not stack 5 variants. Two is the standard. Three is the ceiling.

## When the brief should call it

```
**Campaign Direction**: <one sentence on the constant objective>

**Client Specifications / Additional Notes**: <user must explicitly request variants; include the requested axis (length, voice, urgency)>

**Links**: <verified destination, single URL unless brief carries divergent destinations>

**Framework**: Multi-Variant Draft
```

The brief carries the variant axis the user requested. The copywriter does not invent a third variant when the user asked for two.

## Example structural fragment

```
**Variant A (<ANGLE LABEL, e.g. "Long" or "Founder Voice">)**

Hey [FIRST_NAME],

<COMPLETE VARIANT A BODY>

[<CTA VERB PHRASE>](<verified link from brief>)

<SIGN-OFF>

**Variant B (<MEANINGFULLY DIFFERENT ANGLE LABEL>)**

Hey [FIRST_NAME],

<COMPLETE VARIANT B BODY: different angle, same destination>

[<CTA VERB PHRASE>](<verified link from brief>)

<SIGN-OFF>

[UNSUBSCRIBE_LINK]
```

If the two variants read as paraphrases of each other, the framework has drifted. Either pick the stronger one and drop the framework, or rewrite one variant on a meaningfully different axis.
